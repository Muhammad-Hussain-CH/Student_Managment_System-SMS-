🎯 Project Vision
A web-based Student Management System that allows institutions to manage students, subjects, attendance, exams, fees, and generate reports — all in one place.

👥 User Roles (Who uses the system?)
RoleResponsibilitiesAdminFull control — manage teachers, students, fees, reportsTeacherMark attendance, enter grades, view assigned subjectsStudentView own attendance, results, fee status

📦 Core Modules
1. 🔐 Authentication & Authorization

Login / Logout
Role-based access (Admin / Teacher / Student)
JWT-based sessions
Password reset

2. 👨‍🎓 Student Management

Register / Edit / Delete students
Student profile (name, CNIC, DOB, contact, photo)
Assign to class / section / program
Student ID generation

3. 📚 Subject & Class Management

Create subjects (name, code, credit hours)
Create classes & sections
Assign subjects to classes
Assign teachers to subjects

4. 📅 Attendance Management

Mark daily attendance (Present / Absent / Late / Leave)
Per subject OR per day attendance
Attendance summary per student
Threshold alerts (e.g. below 75%)

5. 📝 Exam & Result Management

Create exams (Midterm, Final, Quiz, Assignment)
Enter marks per student per subject
Auto-calculate GPA / percentage
Grade generation (A, B, C, F)
Result cards / transcripts

6. 💰 Fee Management

Define fee structure (tuition, lab, library, etc.)
Assign fee to students / batches
Track payments (paid / pending / overdue)
Generate fee challan / receipt
Fine for late payments

7. 📊 Reports & Dashboard

Admin dashboard (total students, revenue, attendance %)
Attendance reports (per student, per class, date range)
Result reports (class-wise, subject-wise)
Fee collection reports
Export to PDF / Excel

8. 🔔 Notifications (optional but impressive)

Fee due reminders
Attendance shortage alerts
Exam schedule notifications


🗂️ Suggested Database Entities
Users (id, name, email, password, role)
Students (id, user_id, roll_no, class_id, program, dob, photo)
Teachers (id, user_id, qualification, department)
Classes (id, name, section, program)
Subjects (id, name, code, credit_hours, class_id, teacher_id)
Attendance (id, student_id, subject_id, date, status)
Exams (id, name, type, subject_id, total_marks, date)
Results (id, student_id, exam_id, obtained_marks, grade)
FeeStructure (id, name, amount, class_id)
FeePayments (id, student_id, fee_id, paid_on, status, challan_no)
